Dos paradigmas, no dos versiones

Paradigma 1

La IA sugiere,
vos conducís

El modelo completa una línea, propone un bloque, ofrece una alternativa. Vos abrís el archivo, aceptás o rechazás la sugerencia, ejecutás el test.

La IA opera en modo reactivo: responde cuando la invocás. Vos manejás el flujo.

Paradigma 2

El agente planifica
y ejecuta en loop

Recibe un objetivo, arma un plan, ejecuta pasos, observa resultados, ajusta y sigue.

La IA conduce el flujo; vos supervisás el proceso. El modelo no espera que le traigas el error: va a buscarlo.

El LLM debajo puede ser el mismo. Lo que cambia es la arquitectura encima.

El mapa de la clase

El loop

Qué ciclo ejecuta el agente, qué pasa en cada vuelta, y cómo sabe cuándo parar. Sin entender el loop, no podés anticipar qué hace el agente cuando algo sale mal.

El contexto

El agente trabaja con lo que tiene en su ventana. Eso define qué puede recordar, qué puede razonar, y por qué se pierde en tareas largas.

Pensar — formular el próximo paso

Pensar

El agente razona sobre el estado actual: qué sabe, qué le falta saber, qué hipótesis tiene. No es pensamiento lineal — es el modelo formulando el próximo paso en función de lo que observó antes.

Produce: una decisión sobre qué acción tomar a continuación.

"No es lineal: no sigue un script. El modelo formula cuál es el próximo paso útil dado todo lo que sabe en ese momento. Qué hipótesis tiene, qué le falta ver, qué tiene sentido probar ahora. Lo que produce esta fase es una decisión: qué acción viene a continuación."

Produce: información nueva en el contexto — el insumo de la siguiente vuelta.

ReAct — razonar y actuar entrelazados

Intuición incorrecta

Pensar primero, terminar,
después pasar a actuar.

Como si fueran fases secuenciales:
planificación → ejecución.

Lo que pasa en realidad

Pensar → actuar → observar → pensar → actuar → observar…

Cada observación actualiza el razonamiento.
Pensar y actuar se alternan, vuelta a vuelta.

El nombre: ReAct — Reasoning and Acting.
Lo importante no es el nombre: es que captura que las dos cosas se entremezclan.

LLM

Sin él: no hay razonamiento, el loop no sabe a dónde ir.

Chat

pregunta → respuesta

Vos aportás input. El modelo devuelve output. La sesión puede seguir, pero cada vuelta es independiente. El modelo no actúa entre respuestas — esperás vos.

Agente

objetivo → loop hasta lograrlo

Vos aportás el objetivo. El modelo actúa, observa, corrige, sigue. La sesión no termina hasta que el objetivo se cumple o algo lo interrumpe.

"Arreglá el bug" → secuencia de acciones que termina con el bug arreglado.

El valor entregado no es información sobre el problema: es el problema resuelto.

Condiciones de corte

Objetivo alcanzado

El estado del entorno coincide con lo pedido. Los tests pasan, el endpoint responde 200. Condición ideal.

Límite de pasos

Se alcanzó el techo configurado sin resolver el objetivo. El agente reporta qué hizo y hasta dónde llegó.

Pide ayuda

El agente no puede continuar sin input humano — falta un secreto, no tiene permiso, hay ambigüedad. El loop se suspende.

Falla no recuperable

El agente detecta un estado del que no sabe cómo salir. Para y reporta.

Sin condición de corte, el loop da vueltas. Qué pasa cuando se descontrola, lo vemos en §5.
