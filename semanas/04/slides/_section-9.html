Config del runtime, no instrucciones para el agente

  • Archivo JSON que configura el runtime de Claude Code — el programa que envuelve al LLM.
  • No son instrucciones para el agente. Eso es CLAUDE.md y .claude/rules/. Acá van parámetros del runtime: qué model usar, qué environment variables inyectar, qué puede ejecutar sin pedir confirmación.
  • CLAUDE.md dice "cómo comportarse"; settings.json dice "cómo ejecutar a Claude". Dos canales distintos hacia dos destinatarios distintos.
  • → Una instrucción textual puede ser ignorada o mal interpretada; una regla de runtime no puede.

Nivel 1 Managed
Política de la organización — fija el techo administrado por la org (enterprise) Lo que define un admin para todos los usuarios. Puede bloquear permisos que ningún nivel inferior puede levantar.

Tres cosas que confunden de settings.json

Caso 1

Managed fija un techo, no un default

En equipos enterprise, el admin puede declarar un deny en el nivel Managed que ningún proyecto ni usuario puede levantar. No es un default que vos sobreescribís: es un piso de política que limita lo que cualquier nivel inferior puede pedir. En proyectos personales no aparece; en una org sí.

Caso 2

.local.json al .gitignore, siempre

Es la única forma de que el esquema de niveles funcione bien en equipo. Si commiteás .claude/settings.local.json con un token real, ese token queda en el historial de git — no alcanza con borrarlo en el próximo commit. Hay que rotarlo. El .gitignore del demo-repo ya tiene esa línea desde el primer commit.

Caso 3

No confundir con mcp.json

Los servidores MCP tienen su propio archivo (mcp.json) y su propio mecanismo. No son un campo de settings.json; los dos coexisten en .claude/ sin reemplazarse. Si buscás cómo conectar un MCP server y caés en settings.json, estás en el archivo equivocado.

Recorrer las tres cards en orden. La uno se nombra rápido — relevante en orgs, no en proyectos personales. La dos es la que más vale la pena marcar — si pasa una vez, el token queda en git para siempre. La tres es preventiva: gente que busca configurar MCP y termina editando settings.json.
