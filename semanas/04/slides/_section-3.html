Leer

reversible

Escribir / ejecutar

puede no ser reversible

Editar un archivo, correr git commit, borrar un directorio, llamar un endpoint que muta estado. El entorno quedó en un estado diferente al esperado.

Apertura de §3 — la distinción operativa más importante sobre tools. Referenciada después en los modos de falla y en plan mode. No agregar listas: el contraste vive en la estructura visual.

"Las tools ya las conocen — la mecánica de llamada estructurada / runtime ejecuta / resultado al contexto la vimos en S1 y la ubicamos en el programa externo de §2. Pero hay una distinción operativa que conviene tener presente desde ahora: no todas las tools tienen el mismo peso cuando algo sale mal."

"Una tool que lee no cambia el estado del entorno. Si el agente leyó el archivo equivocado, lo peor que pasó es que llenó el contexto con ruido. Puede intentar otra lectura."

"Una tool que escribe o ejecuta modifica el entorno. Un archivo sobreescrito sin backup. Un git commit que avanzó el historial. Un endpoint que procesó algo. Puede que sea recuperable; puede que no. No hay deshacer automático."

"Por eso el supervisor presta más atención a las tools que actúan sobre el mundo. Esta distinción la vamos a usar más adelante cuando veamos modos de falla y cuando hablemos de permisos."

Cierre de §3 y semilla del §4. El diagrama de acumulación es solo CSS — sin JS. Muestra visualmente que el contexto crece con cada observación. Las tools son cómo el agente toca el mundo; el contexto es donde el agente recuerda qué tocó.

"Hay algo del mecanismo de las tools que conviene ver explícito porque tiene consecuencias para lo que viene."

"Cuando el agente lee un archivo, ese archivo no desaparece después de procesarlo. Queda en el contexto, como parte de la historia de la sesión. Si leyó un archivo de 800 líneas, esas 800 líneas están acá, disponibles para el próximo razonamiento. No es que el agente 'recuerda' vagamente el archivo desde el preentrenamiento — lo tiene en acceso directo."

"Y eso aplica a todo: el output de npm test entró. La respuesta HTTP entró. El directorio listado entró. Vuelta a vuelta, el contexto crece. Las tools son cómo el agente toca el mundo; el contexto es donde el agente recuerda qué tocó. Y ese espacio tiene un tamaño — qué implica eso, viene a continuación."
