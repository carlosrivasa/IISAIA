Slide-cierre del arco. El roadmap del §4 vuelve completo, sin highlight individual, con sub-labels que aterrizan cada skill al TP. Framing: recomendación, no prescripción.

"Para el trabajo final les recomendamos usar este flujo. No hay un modo especial para proyectos integradores; si lo eligen, es el mismo plugin y la misma secuencia que vieron durante la clase."

"Brainstorming les saca el spec del proyecto — no conviene saltearse ese paso, ahí es donde se decide qué construyen. Writing-plans convierte el spec en un plan de tareas chicas. Antes de tocar código, abren la branch con git checkout. Subagent-driven-development corre la implementación delegada — revisan entre tareas, no después de un bloque grande. TDD activa por default — rojo antes que verde. Requesting-code-review corre antes de cerrar el feature. Verification-before-completion corre los comandos finales y cita el output real. Y finishing-a-development-branch les ofrece abrir el PR. El PR es la entrega visible — el repo en GitHub queda como evidencia del proceso entero."

Lo que el flujo NO hace por vos

Qué construir

Decisión tuya, antes de abrir brainstorming. El plugin no inventa el producto.

Si la decisión arquitectónica es la correcta

Decisión tuya, durante brainstorming. Cuando el agente te propone una estructura, vos la aceptás, la corregís o la rechazás.

Si el feature sirve a un usuario real

Decisión tuya, fuera del flujo. Ningún agente sabe si lo que pediste tiene sentido para quien lo va a usar.

Decidir qué entregar

El plugin acelera la ejecución y disciplina el proceso. No decide qué entregar.

Honest disclosure final. Cuatro cosas que el plugin no decide por vos, para que llegues al TP con expectativas alineadas.

"Esto es importante para que no lleguen al trabajo final pensando que el plugin construye el proyecto solo. No lo hace."

"Qué construir es decisión suya, antes de abrir brainstorming. El plugin no inventa el producto."

"Si la decisión arquitectónica es la correcta es decisión suya, durante brainstorming — cuando el agente les propone una estructura, ustedes deciden si la aceptan, la corrigen o la rechazan."

"Si el feature sirve a un usuario real es decisión suya, fuera del flujo. Ningún agente sabe si lo que pidieron tiene sentido para quien lo va a usar."

"El plugin acelera la ejecución y disciplina el proceso. No decide qué entregar. Y eso es lo bueno: les libera tiempo, justamente, para ocuparse de esa parte — que es la única que el plugin no puede hacer."
