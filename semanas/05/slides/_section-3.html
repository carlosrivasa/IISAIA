No clonás. No copiás archivos a mano. Llega por un comando que se apoya en un registro: el marketplace.

"Ya saben qué hay adentro de un plugin: un directorio con piezas conocidas. Pero ese directorio, cuando lo escribió otro autor, vive en su máquina o en su repo. La pregunta operativa es: ¿cómo llega al disco de ustedes?"

Marketplace

Un marketplace es un registro de plugins que Claude Code sabe consultar.

Es el lugar al que le pregunta "¿qué plugins hay y dónde están?" cuando vos escribís un comando de install.

Pre-registrado

claude-plugins-official

Mantenido por Anthropic. Disponible desde el momento en que abrís Claude Code — no hace falta registrar nada.

Comunitarios

cualquier repo de GitHub

Cualquiera puede armar uno. Para usarlo, primero lo registrás:

/plugin marketplace add <usuario>/<repo>

Definición + dos tipos. El oficial viene pre-registrado; los comunitarios se agregan con un comando.

"Un marketplace es un registro de plugins que Claude Code sabe consultar. Es el lugar al que el cliente le pregunta '¿qué plugins están disponibles, y dónde están?' cuando ustedes escriben un comando de install."

"Hay dos tipos. El oficial, claude-plugins-official, lo mantiene Anthropic y viene pre-registrado: no hace falta agregarlo, está disponible desde que abren Claude Code. Y existen los comunitarios: cualquiera puede armar uno, típicamente como repo de GitHub. Para usar uno, primero lo registran con barra plugin marketplace add seguido de usuario barra repo."

"A partir de ese momento, los plugins de ese marketplace están disponibles para instalar igual que los del oficial."

Cuatro comandos cubren el ciclo de vida

/plugin install <plugin>@<marketplace>

Descarga el plugin y registra sus skills, agents y commands. El @<marketplace> desambigua cuando tenés varios registrados.

/plugin update <plugin>

Actualiza a la última versión disponible según el campo version del manifest. Si el manifest no lo declara, cada commit es release nuevo.

/plugin uninstall <plugin>

Desinstala. Por default borra el directorio de datos persistente; pasale --keep-data si vas a reinstalar enseguida.

/plugin list

Lista lo instalado, con versión, marketplace de origen, y si está habilitado o no.

Los 4 comandos del ciclo de vida. Sintaxis exacta + frase corta de qué hace.

"Cuatro comandos cubren todo el ciclo de vida del plugin instalado."

"Install: descarga y registra. La sintaxis es slash plugin install, el nombre del plugin, arroba, el marketplace. Por ejemplo, install formatter at my-marketplace. El arroba marketplace desambigua de qué registro lo están sacando, importante cuando tienen varios registrados."

"Update: actualiza a la última versión disponible según el campo version del manifest que vimos en §2. Si el autor omitió version, Claude Code trata cada commit como release nuevo."

"Uninstall: desinstala. Por default también borra el directorio de datos persistente. Si querés conservarlo — por ejemplo porque vas a reinstalar una versión nueva en seguida — agregás --keep-data."

"List: muestra lo que tenés instalado, con versión, marketplace de origen, y si está habilitado o no. Es el comando que van a usar primero para verificar instalaciones."

El scope decide quién ve el plugin

Scope
Dónde se registra
Cuándo usarlo
userdefault
~/.claude/settings.json
Uso personal. Te sirve para todos tus proyectos.
project
.claude/settings.json del repo
Plugins de equipo. Queda versionado — viaja con el repo.
local
.claude/settings.local.json (gitignored)
Cosas que sólo querés en tu copia, no en la del equipo.

Tres scopes con cuándo usar cada uno. Default es user; el opt-in interesante es project.

"La doc define cuatro scopes, pero alcanza con tener claros los tres que usan en la práctica."

"Por default, install va a user: el plugin queda registrado en .claude/settings.json en su home, y les sirve para todos sus proyectos. Es el caso más común."

"Si pasan --scope project, va al .claude/settings.json del repo en el que están parados. Queda versionado y se comparte con cualquiera que clone — útil para plugins de equipo. Por ejemplo, ejecutan slash plugin install formatter at my-marketplace --scope project y el plugin viaja con el repo."

"Si pasan --scope local, queda en .claude/settings.local.json, que está gitignored, para cosas que quieren sólo en su copia del proyecto. Default user, opt-in a project cuando quieren que el plugin viaje con el repo."
